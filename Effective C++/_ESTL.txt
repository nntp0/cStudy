23. 연관 컨테이너 대신, 정렬된 vector 가 더 나은 선택일 수도 있다.
    
    - 빠른 데이터 검색을 위해서 가장 효과적인 데이터 구조는 아무래도 해쉬 컨테이너.
    
    - 표준 연관 컨테이너는 일반적으로 균형 이진 탐색 트리.
    
    - 표준 연관 컨테이너보다 ordered vector 가 나은 이유
        1) 데이터 메모리 : 하나의 데이터를 표현하기 위해 오버헤드가 존재.
        2) locality : 데이터를 표현하기 위해 더 큰 용량을 요구하므로, 페이지 하나에 들어갈 수 있는 데이터의 개수가 적게 됨. 이는 page fault 로 이어질 수 있다. 더 나아가, 연관 컨테이너는 node 기반 데이터 구성을 갖고 있어서, 순차 기반 데이터 구성에 비해 데이터가 곳곳에 흩뿌려져 있을 것이며 이는 locality 의 장점을 살리지 못하게된다.

        * 만약, vector 로 map 을 대체하게 되는 경우, vector 가 갖는 요소의 const 한정자는 제거해야한다.
